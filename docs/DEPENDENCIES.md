# Dependencies — Dodona

## Cargo dependencies

All crates pinned in `Cargo.toml`. Versions are the latest stable as of
project start. We update deliberately, not on a schedule.

| Crate | Version | Purpose | Why this one |
|---|---|---|---|
| `smithay-client-toolkit` | 0.19 | Wayland client + layer-shell | De facto standard for Rust Wayland clients |
| `tiny-skia` | 0.11 | CPU 2D rendering (paths, gradients, text, blur) | Skia-quality, no GPU dependency, deterministic |
| `cosmic-text` | 0.12 | Text shaping and layout | Handles complex scripts, ligatures, fallback fonts |
| `zbus` | 5 | DBus (NetworkManager, WirePlumber, MPRIS, notifications) | Typed, async, native tokio integration |
| `tokio` | 1 | Async runtime | Pairs with zbus |
| `notify` | 6 | File watches on /proc, /sys | Cross-platform, simple API |
| `nix` | 0.28 | Unix syscalls (signals, fds) | Idiomatic Rust wrapper |
| `serde` | 1 | Serialization | Standard |
| `toml` | 0.8 | Config parsing (mango config, mmsg output) | Standard |
| `anyhow` | 1 | Error handling at boundaries | Standard |
| `thiserror` | 1 | Error types in library modules | Standard |
| `tracing` | 0.1 | Structured logging | Async-aware, zero-cost when disabled |
| `tracing-subscriber` | 0.3 | Log output | Pairs with tracing |
| `maxminddb` | 0.24 | GeoIP lookup (GeoLite2) | Pure Rust reader, no C deps |
| `rodio` | 0.19 | Audio playback | Handles pipewire/PulseAudio auto, preloaded buffers |
| `crossbeam-channel` | 0.5 | Sync channels for Wayland ↔ async | Lower latency than tokio mpsc for UI events |

Total transitive deps: ~120 crates (typical for a Rust app of this scope).
Release binary size estimate: 12–15MB after `strip` and `lto = true`.

## GeoIP database

- **Database:** MaxMind GeoLite2 Country (free, requires account)
- **Size:** ~6MB compressed, ~60MB uncompressed
- **Location:** `dodona/assets/geoip/GeoLite2-Country.mmdb`
- **Update:** systemd timer weekly, fetches latest from MaxMind
- **License:** MaxMind End User License Agreement (free for personal use,
  requires registration)
- **Fallback:** If DB missing or lookup fails, globe shows abstract
  topology (no geo, just traffic volume as line thickness from center)

### Initial DB setup

1. Register at https://www.maxmind.com/en/geolite2/signup
2. Sign in, generate a license key
3. Store the key in `~/.config/dodona/maxmind-key` (NOT committed to repo)
4. The Nix module reads this file at build time and fetches the DB
5. Weekly systemd timer re-fetches to keep it current

If the key file is missing, dodona builds and runs in fallback mode
(no geo, abstract topology globe). The build does not fail.

## Build dependencies

- Rust 1.75+ (stable channel)
- `pkg-config` (only if any transitive C dep needs it — most don't)
- Nix handles all of this — `nix build .#dodona` does everything

## What we explicitly do NOT depend on

- GTK (any version)
- Qt
- Electron / Node.js / Chromium
- TypeScript / JavaScript engines
- OpenGL / Vulkan (CPU rendering only)
- X11 / XWayland (Wayland only — apps that need XWayland still work, but
  dodona itself never speaks X11)
- PulseAudio (we use pipewire directly via rodio)
- D-Bus Glib (we use zbus, pure Rust)
- libcurl (we use `reqwest` only if needed; for MaxMind fetch we use
  Nix's `fetchurl` instead)

## Dependency review policy

Before adding any new crate:

1. Is it maintained? (last commit < 1 year ago)
2. Is it widely used? (>1000 stars OR <5 reverse deps is a smell)
3. Does it pull in heavy transitive deps? (run `cargo tree` and check)
4. Is there a lighter alternative?
5. Is the license compatible? (MIT, Apache-2.0, MPL-2.0, BSD — yes;
   GPL — only if we're willing to GPL dodona; anything else — no)

If any answer is "no" or "unclear", we discuss before adding.
